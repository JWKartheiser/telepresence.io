Have you ever wanted the quick development cycle of local code while still having your code run within a remote Kubernetes cluster? Telepresence allows you to develop locally, running your code on your machine, while still making your code appear as if it is running in Kubernetes.

  • Your local process can talk to Kubernetes Services and cloud databases like AWS RDS even if they're in a private VPC.
  • Your local process has the same environment variables and Kubernetes volumes as the real pod.
  • The Kubernetes cluster can talk to your local process, which means other Services, your LoadBalancer or your Ingress can send queries to the local process you are running.
